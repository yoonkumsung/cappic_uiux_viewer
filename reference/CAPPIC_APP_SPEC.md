# Cappic - AI 스포츠 자동 촬영 & 하이라이트 시스템

## 앱 기능 명세서 v1.0

---

## 1. 시스템 개요

Cappic은 **축구/풋살 경기를 자동으로 촬영하고, AI가 선수별 하이라이트 영상을 자동 생성하는 시스템**이다.

### 시스템 구성

| 구성 요소 | 역할 | 기술 스택 |
|-----------|------|-----------|
| **cappic-camera** | RK3588 기반 엣지 카메라 장치 (YOLO 추론, 모터 제어, GStreamer 녹화, WebRTC 스트리밍) | Python, GStreamer, RK3588 NPU |
| **cappic-detector** | 사람 + 공 객체 탐지 AI 모델 (카메라 장치에 배포) | PyTorch → ONNX → RKNN INT8 |
| **cappic-highlight** | 선수별 볼 액션 탐지 + 하이라이트 자동 생성 AI | PyTorch, DINOv3, YOLO, BoxMOT |
| **cappic-monitor** | 카메라 관제/녹화/하이라이트 생성 앱 (현재 Flutter) | Flutter, Riverpod, Drift, WebRTC |
| **스케줄 서버** | 촬영 스케줄 관리 API 서버 | Node.js (192.168.0.155:3000) |

### 데이터 흐름

```
[카메라 장치 (RK3588)]
  ├── YOLO 실시간 객체 탐지 (사람 + 공)
  ├── 모터 자동 패닝 (공/선수 추적)
  ├── 4K 녹화 (GStreamer → 세그먼트 → 클라우드 업로드)
  ├── WebRTC 저지연 프리뷰 (960x540, 15fps)
  └── ReID 세션 처리 (인물 선택 → 개인 하이라이트 렌더)
          ↓
[관제 앱 (cappic-monitor)]
  ├── 카메라 탐색/모니터링/녹화 제어
  ├── 영상 아카이브 관리
  ├── ReID 하이라이트 워크플로우
  └── 촬영 스케줄 관리
          ↓
[AI 하이라이트 파이프라인 (cappic-highlight)]
  ├── YOLO + BoxMOT 선수 추적
  ├── DINOv3 + TCN 액션 감지 (9개 클래스)
  ├── 하이라이트 패턴 매칭 (Drive→Shot 등)
  └── 자동 클립 생성 + 메타데이터
```

## 2. 앱 분리 계획

현재 `cappic-monitor`는 관리자 전용 관제 앱이다. 이를 **관리자용 앱**과 **소비자용 앱**으로 분리한다.

| 구분 | 관리자용 앱 (Admin) | 소비자용 앱 (Consumer) |
|------|--------------------|-----------------------|
| 대상 | 촬영 감독, 시설 운영자, 코치 | 선수, 학부모, 팬 |
| 핵심 가치 | 촬영 시스템 관제 & 운영 | 나만의 하이라이트 보기 & 공유 |
| 기존 기능 | cappic-monitor 전체 | 일부 (아카이브, ReID, 비디오 재생) |
| 신규 기능 | 고급 관제 기능 확장 | SNS형 경험, 개인화, 공유 |

## 3. 관리자용 앱 (Cappic Admin)

### 3.1 카메라 탐색 & 관리

#### 3.1.1 자동 카메라 탐색 (Discovery)
- 로컬 WiFi /24 서브넷 스캔 (192.168.x.1~254)
- TCP 소켓 연결 확인 (포트 8000, 100ms 타임아웃)
- HTTP `/api/v1/ping` 헬스 체크
- 배치 25개씩 병렬 스캔 (약 1초에 254개 IP 완료)
- 진행률 표시 (스캔 진행/완료 IP 수)
- 발견된 카메라에 자동 ID 할당 (`setCameraId`)
- SQLite에 탐색 이력 저장 (시간, 발견 카메라 수)

#### 3.1.2 카메라 목록
- 발견된 카메라 카드 리스트
- 카메라별 표시 정보: 이름, IP, 카메라 ID, 그룹 ID, 녹화 상태
- **카메라 이름 변경** (롱프레스 → 다이얼로그)
- **카메라 ID / 그룹 ID 설정** (서버 API 연동)
- 마지막 탐색 시간 표시
- Pull-to-refresh로 재탐색 트리거

#### 3.1.3 WiFi 상태 모니터링
- 연결 상태, SSID, 로컬 IP 실시간 표시
- WiFi 미연결 시 경고 배너
- 연결 변경 스트림 자동 구독

#### 3.1.4 네트워크 디버그 페이지
- 모든 네트워크 인터페이스 나열
- 선택된 IP, 감지된 서브넷 표시
- 사설 IP 필터링 (Android rmnet 제외)
- 클립보드 복사 기능

### 3.2 실시간 카메라 모니터링

#### 3.2.1 WebRTC 실시간 프리뷰
- VP8 코덱, 960x540, 15fps 저지연 스트리밍
- SDP offer/answer 교환 + ICE candidate 처리
- 자동 재연결 (최대 5회, 지수 백오프)
- 연결 상태 표시 (connecting/connected/disconnected/failed)

#### 3.2.2 세로 모드 카메라 상세
- WebRTC 실시간 영상 프리뷰
- REC 필(맥동 애니메이션) + 녹화 시간 뱃지
- 설정 요약 행: RESOLUTION, STORAGE, CPU
- START / STOP RECORDING 버튼
- 풀스크린 전환 버튼 → 가로 전체 화면

#### 3.2.3 가로 모드 전체 화면 프리뷰
- 강제 가로 모드 + 몰입형 UI (시스템 바 숨김)
- WebRTC 전체화면 스트리밍
- 탭으로 오버레이(컨트롤) 토글
- ESC 키 지원 (데스크톱)
- 사이드 메트릭 드로어 (360px 슬라이드-인)

#### 3.2.4 대시보드 그리드 뷰 (레거시)
- 다중 카메라 프리뷰 그리드
- 반응형 컬럼 수 자동 조정
- 다중 선택 + 일괄 촬영 제어

### 3.3 녹화 제어

#### 3.3.1 개별 녹화 제어
- 카메라별 녹화 시작/중지
- 녹화 상태 실시간 표시 (5초 자동 폴링)
- 녹화 중지 → 저장 중 오버레이 (SpinRing 애니메이션)
- 저장 완료 → 파일명, 녹화 시간 확인 + 아카이브/홈 이동

#### 3.3.2 다중 선택 배치 녹화
- 체크박스로 여러 카메라 선택
- 선택한 카메라 일괄 녹화 시작/중지
- **낙관적 UI 업데이트**: 중지 요청 즉시 UI 반영, API 실패 시 롤백
- 배치 제어 중 개별 오류 처리 (실패한 카메라만 표시)

### 3.4 종합 메트릭 대시보드

#### 3.4.1 실시간 메트릭 (10초 자동 갱신)

| 메트릭 카테고리 | 세부 항목 |
|----------------|----------|
| **녹화 상태** | 활성 여부, 시작 시간, 경과 시간, 세션 디렉토리, 세션 ID |
| **업로드 상태** | 총/완료/대기/실패 세그먼트 수, 업로드 속도 (Mbps), 진행률 |
| **시스템 리소스** | CPU 사용률, 메모리 사용률, 온도, NPU 부하, 디스크 사용량(사용/전체/퍼센트), GStreamer 프로세스 상태 |
| **카메라 설정** | V4L2 하드웨어 파라미터 13개 (gain, exposure, brightness, contrast, saturation, hue, gamma, sharpness, backlight_compensation, white_balance_temperature, focus, zoom, auto_gain) |
| **추적 정보** | YOLO 모델 로드 상태, 추론 시간(ms), 감지된 공 수, 감지된 사람 수, 모터 각도/방향 |
| **건강 상태** | healthy / degraded / error 3단계 판정, 경고 메시지, 에러 메시지 |

#### 3.4.2 메트릭 퀵 로우
- REC (녹화 시간), CPU (사용률), TEMP (온도), DET (감지 수) 칩 요약

#### 3.4.3 히스토리 메트릭 차트
- 날짜 범위 또는 세션 ID 기준 과거 메트릭 조회
- 60초 간격 수집, 30일 보관
- **차트 종류**:
  - `MetricsLineChart` - 단일 라인 (CPU, 온도 추이)
  - `MetricsMultiLineChart` - 다중 라인 비교
  - `MetricsBarChart` - 바 차트
  - `MetricsPieChart` - 파이 차트 (디스크 사용량 비율)

### 3.6 영상 아카이브 관리

#### 3.6.1 경기 영상 탭
- 녹화 원본 (recording) 목록
- 업로드된 영상 (uploaded) 목록
- 파생 영상 (derived - concat 결과 등) 목록
- 영상별 정보: 파일명, 날짜, 파일 크기, 해상도

#### 3.6.2 하이라이트 탭
- AI 생성 하이라이트 목록
- ReID 결과 영상 목록
- 진행 중인 ReID 세션 배너 (렌더 대기/진행 중 표시)

#### 3.6.3 비디오 업로드
- 기기에서 비디오 파일 선택 (file_picker)
- 카메라 서버에 multipart 업로드
- 업로드 진행률 + 속도 (MB/s) 실시간 표시
- **2GB 업로드 크기 제한** 검증

#### 3.6.4 비디오 연결 (Concat)
- 여러 클립 선택 → 서버에서 concat job 생성
- 작업 상태 폴링 (job ID 기반)
- 완료 후 결과 영상 다운로드 가능

#### 3.6.5 비디오 재생
- 로컬 파일 / HTTP URL 재생 (video_player)
- 재생/일시정지, 10초 앞/뒤 탐색
- 스크럽 바 (Slider) + 시간 표시 (현재/전체)
- 풀스크린 전환 (가로 모드, 동일 컨트롤러 공유 → 버퍼링 없이 이어짐)

#### 3.6.6 파일 다운로드
- 스트림 기반 다운로드 (청크 타임아웃, 진행률 표시)
- 다운로드 완료 후 시스템 갤러리 자동 저장 (gal 패키지)
- 플랫폼별 다운로드 디렉토리 자동 결정

### 3.7 ReID 하이라이트 워크플로우

#### 3.7.1 워크플로우 3단계

**Step 1: 소스 선택 (MediaSourcePicker)**
- 서버 녹화본에서 비디오 선택
- 기존 ReID 세션 이어서 진행
- 여러 클립 concat 후 사용
- 기기에서 비디오 업로드 (multipart, progress 콜백)

**Step 2: 인물 선택**
- 타임라인 슬라이더로 영상 내 시간 탐색
- 프레임 캔버스에 YOLO 검출 바운딩 박스 표시
- 검출된 인물 썸네일 그리드 (DetectionThumbnailGrid)
- 인물 탭 → 추적 대상 선택 (bbox 좌표 전달)
- **다중 인물 축적 선택** 가능 (PersonChip으로 선택된 인물 표시)

**Step 3: 출력 설정 & 렌더**
- score_threshold 조절 (Slider)
- match_threshold 조절 (Slider)
- output_fps 설정
- bitrate 설정
- debug 모드 토글
- **배치 제출**: 선택된 인물 수만큼 개별 ReID 세션 자동 생성

#### 3.7.2 렌더 진행 모니터링 (HighlightProgressPage)
- 다중 ReID 세션 배치 렌더 동시 진행
- 2초 간격 폴링
- 세션별 진행률 바 + 퍼센트 표시
- 현재 처리 중인 세션 ID 표시
- "백그라운드로 진행" 버튼 (팝 후 스낵바 알림)

#### 3.7.3 렌더 완료 (HighlightCompletePage)
- 바운스-인 체크 애니메이션
- 세션별 결과 리스트 (성공/실패 표시)
- "아카이브에서 보기" / "홈으로" 이동

#### 3.7.4 ReID 세션 관리
- 카메라별 ReID 세션 전체 목록
- 세션 상태: pending / processing / completed / failed
- 완료된 세션 결과 비디오 다운로드 + 갤러리 저장
- 세션 삭제 (확인 다이얼로그)

## 4. 소비자용 앱 (Cappic Consumer)

### 4.0 사용자 유형 & 진입 경로

소비자 앱은 **Owner**와 **Visitor** 두 가지 사용자 유형을 하나의 앱에서 지원한다.

| | Owner (카메라 소유자) | Visitor (구장 이용자) |
|---|---|---|
| **진입 경로** | 카메라 구매 → 앱 설치 → 카메라 연결 | 구장 경기 → QR/링크 → **웹 미리보기** → 앱 설치 유도 |
| **핵심 니즈** | 촬영 제어 + 영상 관리 + 하이라이트 | 내 경기 영상 찾기 + 하이라이트 + 공유 |
| **카메라 권한** | 연결/녹화/설정 모두 가능 | 없음 (영상 소비만) |
| **하단 탭** | 촬영 · 영상 · 하이라이트 · 마이 | 영상 · 하이라이트 · 마이 |

#### 4.0.1 온보딩 분기

```
┌──────────────────────────────────┐
│         Cappic 시작하기           │
│                                  │
│  ┌────────────┐  ┌────────────┐  │
│  │   📹       │  │   ⚽       │  │
│  │  카메라    │  │  경기      │  │
│  │  가지고    │  │  영상만    │  │
│  │  있어요    │  │  볼래요    │  │
│  └────────────┘  └────────────┘  │
│                                  │
│  "나중에 카메라를 구매하면        │
│   언제든 전환할 수 있어요"       │
└──────────────────────────────────┘
```

#### 4.0.2 Visitor 웹 프리뷰 플로우 (앱 설치 전)

구장에서 경기 후, 앱을 설치하지 않은 사용자도 **웹 브라우저에서 즉시 하이라이트를 미리볼 수 있는** 경로:

```
경기장 QR 포스터 또는 링크 공유
  ↓
웹 브라우저 (앱 설치 불필요)
  ↓
구장 페이지 → 오늘 경기 목록
  ↓
경기 선택 → 1분 하이라이트 720p 미리보기 + 다운로드
  ↓
"더 많은 하이라이트와 경기 영상을 보려면 앱을 다운받으세요"
  ↓
앱스토어/플레이스토어 유도 → 앱 설치 → 가입
  ↓
웹에서 보던 구장/경기가 앱에 자동 연결
```

**웹 미리보기에서 제공하는 것:**
- 해당 경기의 **1분 하이라이트 720p** 재생 & 다운로드
- 인물별 썸네일 그리드 (앱에서만 상세 보기 가능하다는 안내)
- 구장 정보, 경기 시간

**웹에서 제공하지 않는 것 (앱 유도):**
- 10분 하이라이트
- 전체 경기 영상
- 개인 ReID 하이라이트
- 4K 다운로드
- 인물별 개별 하이라이트 재생

```

│  ── 선수별 하이라이트 ──   │
│  ┌────┐┌────┐┌────┐      │
│  │ 😊 ││ 😎 ││ 🧑 │      │
│  │    ││    ││    │      │
│  └────┘└────┘└────┘      │

│  └────┘└────┘└────┘      │
│  ┌────┐┌────┐┌────┐      │

│  └────┘└────┘└────┘      │
│                          │
└──────────────────────────┘
```

**영상 3종:**
| 영상 | 길이 | 설명 |
|------|------|------|
| 전체 영상 | 원본 길이 (40~90분) | 경기 전체 녹화본 |
| 1분 하이라이트 | ~1분 | AI가 뽑은 핵심 장면 (골, 슛, 세이브) |
| 10분 하이라이트 | ~10분 | 주요 플레이 전체 (드리블, 패스, 크로스 포함) |

**선수별 하이라이트 그리드 (인스타그램 릴스 스타일):**

- **3x3 (또는 NxN) 그리드 레이아웃** — 경기 참여 인원수만큼 표시 (10명이면 10칸)
- **각 칸 = 해당 선수의 크롭 썸네일**
  - AI가 경기 영상에서 해당 인물의 얼굴/체형이 **가장 높은 정확도로 잡힌 프레임**에서 크롭
  - 정사각형 크롭 (인물 중심)
  - 등번호 뱃지 표시 (감지된 경우)
- **프레스 앤 홀드 → 5초 미리보기 재생**
  - 해당 선수의 베스트 하이라이트 5초 미리보기
  - 인스타그램 릴스 미리보기와 동일한 인터랙션
  - 손 떼면 정지, 썸네일로 복귀
- **탭 → 해당 선수 개인 하이라이트 전체 재생**
  - 해당 선수의 모든 액션 (Drive, Pass, Shot 등) 모아보기
  - 다운로드/공유 가능

```
인터랙션 흐름:
[썸네일 그리드] → (프레스 앤 홀드) → [5초 미리보기 재생]
                → (탭)           → [개인 하이라이트 전체 페이지]
                                    ├── 재생
                                    ├── 다운로드
                                    └── 공유 (카카오톡, 인스타, 링크)
```

## 5. AI 파이프라인 상세 (백엔드 참고)

### 5.1 객체 탐지 (cappic-detector)

| 항목 | 상세 |
|------|------|
| 탐지 대상 | 사람 (person) + 공 (ball) 2클래스 |
| 백본 | MobileNetV4 Conv Small (9M 파라미터 이하) |
| 넥 | LitePAN (3레벨 FPN) |
| 헤드 | LightweightDetectionHead (Depthwise Separable Conv) |
| 특징 | NMS-free 배포 (one-to-one head) |
| 입력 | 640x640 고정 |
| 배포 | PyTorch → ONNX (opset 19) → RKNN INT8 양자화 |
| 타겟 | RK3588 NPU 3코어 병렬 실시간 추론 |
| 타일링 | 와이드 이미지 4:3 비율 동적 타일 분할 |
| 학습 데이터 | COCO 2017 (person + sports ball) + Open Images V7 (Football) |
| person threshold | 0.60 |
| ball threshold | 0.50 |

### 5.2 액션 감지 (cappic-highlight)

| 항목 | 상세 |
|------|------|
| 아키텍처 | Teacher-Student Knowledge Distillation |
| Teacher 백본 | DINOv3 ViT-L/16 (frozen, 1024-dim) |
| Student 백본 | DINOv3 ConvNeXt-Tiny (768-dim) |
| 헤드 | TCN 4 layers (dilation 1,2,4,8), 수용 필드 65프레임 (4.33s) |
| 전술 피처 | 22차원 MLP (attack_x/y, speed, dist_to_goal, angle, role 등) |
| 액션 클래스 | 9개 (Background, Drive, Pass, Cross, Throw-in, Shot, Header, Tackle, Block) |
| 작업 프레임레이트 | 15fps (25fps SoccerNet / 30-60fps 풋살 통일) |
| 윈도우 | 4초, 60프레임 |
| 크롭 | 선수별 224x224 (bbox 1.8배 확장) |
| KD 손실 | Task Loss (α=1.0) + Logit KD (β=2.0, τ=4.0) + Temporal KD (δ=0.5) |
| 평가 메트릭 | FOOTPASS F1@delta (delta=12 frames @25fps, ~0.5초 허용) |
| 선수 추적 | YOLO11x + BoxMOT (BotSort + OSNet ReID) |
| 배포 타겟 | RK3588 NPU (Student ONNX 내보내기) |

### 5.3 하이라이트 생성 파이프라인

```
1. YOLO11x 선수/공 탐지
2. BoxMOT BotSort+OSNet 다중 선수 추적
3. 선수별 크롭 → DINOv3 BF16 피처 추출
4. 60프레임 버퍼 → TCN 슬라이딩 윈도우 추론 (stride=4)
5. ScoreAccumulator 프레임별 점수 누적
6. Action Segmentation (conf>=0.4, min 2프레임, merge gap 3)
7. Cross-player 병합 (트랙 ID 스왑 보정)
8. 하이라이트 패턴 매칭 (7개 명명 패턴 + 일반 마무리 규칙)
9. 클립 경계 계산 (pre 1.5초, post 0.5초, min 3초, max 15초)
10. 시간적 중복 제거 (같은 클래스 겹침 → ball 가장 가까운 선수 유지)
11. bbox 오버레이 MP4 내보내기 + ffmpeg H.264 재인코딩
12. actions.json / highlights.json / timelines.json 메타데이터 생성
```

## 12. 관리자 UX 설계 원칙 (IT 초등학생 수준 대응)

### 12.1 핵심 원칙

| 원칙 | 가이드 |
|------|--------|
| **큰 터치 타겟** | 최소 44x44px (9~12mm), 요소 간 5mm+ 패딩 |
| **한 화면 한 작업** | 화면당 하나의 주요 목표만, 메뉴 깊이 2~3단계 제한 |
| **즉각적 피드백** | 터치 시 색상 변화/짧은 애니메이션, 성공/오류 메시지 즉시 표시 |

| **자동화 극대화** | 스케줄 자동 녹화, 자동 하이라이트, 자동 클라우드 업로드 — 관리자 개입 최소화 |
| **오류 방지** | 잘못된 입력 사전 차단, 실수 시 쉬운 "되돌리기" |

## 13. 소비자 앱 상세 설계 (경쟁사 분석 반영)

### 13.1 MVP (v1.0) 필수 기능

경쟁사 공통 제공 기능 + Cappic 차별화 기능:

#### A. 카메라 연결 & 제어 (XBOTGO/Veo 참고)
- 원탭 자동 연결 (디바이스 자동 검색)
- 원격 녹화 시작/중지 (2탭 이내)
- 실시간 카메라 프리뷰
- 배터리/저장공간 상태 표시

#### B. AI 자동 하이라이트 (Trace AI Reels 참고)
- **원탭 하이라이트 릴 생성** (경기 후 자동)
- 9가지 볼 액션 기반 핵심 장면 감지 (Cappic만의 강점)
- 하이라이트 패턴 매칭 (Drive→Shot, Drive→Cross→Header 등)
- 템플릿 + 배경음악 자동 적용
- SNS 비율 자동 변환 (세로/정사각/가로)

#### C. 개인 선수 하이라이트 (Veo Player Spotlight / Trace PlayerFocus 참고)
- **ReID 기반 "나 찾기"** (Cappic만의 강점 — 경쟁사는 등번호 의존)
- 인물 선택 → 개인 하이라이트 자동 생성
- 렌더 진행 실시간 표시
- 완료 후 즉시 재생 & 저장

#### D. 영상 관리
- 경기 영상 라이브러리 (날짜/팀별 정렬)
- 클라우드 업로드/다운로드 (최소 10~20GB 무료)
- 비디오 재생 (배속 조절, 풀스크린)
- 기본 트리밍/클립 생성

#### E. 공유 (XBOTGO/Trace 참고)
- SNS 직접 공유 (카카오톡, 인스타그램, TikTok, YouTube)
- 공유 링크 생성
- QR 코드 공유
- 기기 갤러리 저장

#### F. 라이브 스트리밍 (XBOTGO 참고)
- YouTube/Facebook 라이브 연동
- 공유 가능한 스트리밍 링크
- 실시간 스코어보드 오버레이

#### G. 기본 UX
- 초보자 친화 온보딩 (인터랙티브 튜토리얼)
- 3탭 이내 주요 기능 접근
- 다국어 (한/영/일)
- 네이버/카카오 소셜 로그인

### 13.2 v2.0 확장 기능

#### A. 통계 & 분석 (Veo Analytics 참고)
- 히트맵 (활동 영역 시각화)
- 기본 매치 통계 (점유율, 슛, 패스)
- 선수별 통계 대시보드
- 경기 모멘텀 그래프
- AI 코칭 어시스턴트 (경기 후 자동 분석 요약 + 개선 포인트)

#### B. 팀 관리 (Trace/TeamSnap 참고)
- 팀 생성/관리 (로스터, 등번호, 포지션)
- 팀원 초대/역할 구분 (코치/선수/학부모)
- 팀별 영상 라이브러리
- 코치 메모/드로잉 도구 (텔레스트레이션)
- 학부모 뷰 (자녀 하이라이트 자동 수신)

#### C. 고급 편집 (Veo Editor 참고)
- 슬로우 모션 / 프레임별 분석
- 사이드바이사이드 비교
- 커스텀 그래픽/워터마크
- 배경음악/효과음 라이브러리

#### D. 리크루팅/프로필 (Trace iD / Hudl 참고)
- 선수 프로필 페이지 (하이라이트 모음)
- 대학/스카우트 공유 최적화
- 선수 포트폴리오 PDF 내보내기

#### E. 세컨드 스크린 (XBOTGO 참고)
- 학부모/팬용 실시간 시청 (별도 경량 뷰어)
- 실시간 채팅/응원

### 13.3 Cappic 차별화 전략

| 전략 | 상세 | 경쟁 포지셔닝 |
|------|------|-------------|
| **무구독 + 프리미엄 하이브리드** | 핵심 기능 무료 + 고급 분석/통계 $5~10/월 | Veo ($67~109/월) vs XBOTGO (완전 무료) 사이 |
| **ReID 기반 선수 추적** | 등번호 불필요, AI 인물 재식별로 정확한 개인 추적 | Veo/Trace는 등번호 의존 → 유니폼 없으면 불가 |
| **9개 액션 클래스 감지** | Drive/Pass/Cross/Shot/Header/Tackle/Block/Throw-in 자동 감지 | 경쟁사는 "하이라이트 자동 생성"만, 액션 분류 없음 |
| **RK3588 엣지 AI** | 카메라 장치에서 실시간 AI 추론 → 서버 의존도 낮음 | 경쟁사 대부분 클라우드 의존 |
| **풋살 특화** | 실내 풋살/축구교실 최적화 (와이드 코트 타일링, 15fps 통일) | 경쟁사 대부분 축구 11인 필드 기준 |
| **한국 시장 네이티브** | 카카오/네이버 로그인, 한글 UX, 축구교실/학부모 문화 반영 | 경쟁사 전부 해외 (한국어 미지원 or 부분 지원) |
| **SNS 네이티브 하이라이트** | TikTok/Reels 최적화 자동 편집, 세로/정사각/가로 변환 | 경쟁사는 가로 영상 위주 |
