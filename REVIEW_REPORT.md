## 1. 사용자 경험 (UX)

### 1.1 핵심 플로우 끊김

현재 프리뷰에서 **유저가 앱을 처음 접했을 때의 흐름**이 자연스럽게 연결되지 않는 구간이 있다.

#### 관리자 앱

| 끊기는 지점 | 문제 | 해결 |
|------------|------|------|
| 로그인 → 홈 | 최초 로그인 후 카메라가 하나도 없을 때의 Empty State 없음 | "카메라를 찾아볼까요?" 빈 상태 화면 + 자동 검색 안내 추가 |
| 홈 → 카메라 상세 | 어떤 카메라를 눌러도 같은 상세 화면으로 이동 | 카메라별 데이터 바인딩 구분 필요 (최소 이름이라도) |
| 아카이브 → 영상 플레이어 | 아카이브 목록에서 영상을 탭해도 어디로 가는지 불명확 | 탭 → ADM-10L(영상 플레이어)로 이동하는 `data-go` 연결 필요 |
| 촬영 관리(ADM-02) → 저장 중(ADM-05) | "녹화 중지" 버튼이 보이지 않음. REC 상태에서 중지하는 방법 불명 | 녹화 중일 때 "녹화 중지" 버튼 노출 필요 |
| AI 하이라이트 워크플로우 | Step 3 → 생성 중 → 완료 후 아카이브로 돌아가는 흐름이 약함 | 완료 화면에서 "아카이브에서 보기" 버튼 연결 확인 |

#### 소비자 앱

| 끊기는 지점 | 문제 | 해결 |
|------------|------|------|
| 온보딩(CON-01) → 홈(CON-04) | 프로필 설정(CON-03) 후 바로 홈으로 이동하는데, **구장을 추가하는 단계가 없음** | 프로필 설정 완료 → "구장 추가" 화면 → 홈 순서 필요 |
| 홈(CON-04) → 경기 목록(CON-06) | 홈에서 구장 카드를 탭하면 경기 목록으로 가야 하는데 연결이 약함 | 구장 카드에 `data-go="con-06"` 확실히 연결 |
| 영상 상세(CON-07) → 선수 탭 | 선수 썸네일을 탭했을 때 어디로 가는지 불명 | 개인 하이라이트 상세 화면(스펙 §4.2.3) 필요 |
| 피드(CON-FD) | 피드 화면은 있지만, 어떤 경로로 콘텐츠가 채워지는지 불명 | 피드 콘텐츠 소스와 알고리즘 기획 필요 |

### 1.2 피드백 & 상태 전달 부족

| 상황 | 현재 | 개선 |
|------|------|------|
| 녹화 시작/중지 | 즉시 반영되는 것처럼 보이지만 확인 피드백 없음 | 토스트 알림: "녹화를 시작했어요" / "녹화를 중지했어요" |
| 카메라 연결 끊김 | 카드 상태 도트만 변경 | **배너 알림**: "B구장 2번 카메라 연결이 끊어졌어요" + 진동/사운드 |
| 파일 다운로드 완료 | 별도 표시 없음 | 토스트: "갤러리에 저장했어요" |
| AI 하이라이트 생성 실패 | 실패 상태만 표시 | **왜 실패했는지** 쉬운 말로 설명 + 재시도 버튼 |
| 로딩 상태 | 일부 화면에 스피너 있지만, 대부분의 데이터 로딩에 스켈레톤 없음 | 카메라 목록, 아카이브, 경기 목록 등에 **스켈레톤 UI** 적용 |

### 1.3 접근성 문제

| 문제 | 영향 | 해결 |
|------|------|------|
| `<button>` 대신 `<div>` + `cursor:pointer` 사용하는 곳 다수 | 키보드 탐색 불가, 스크린 리더 미인식 | 모든 클릭 가능 요소를 시맨틱 `<button>` 또는 `<a>` 로 변경 |
| `aria-label` 없음 | SVG 아이콘 버튼의 용도를 스크린 리더가 알 수 없음 | 모든 아이콘 버튼에 `aria-label` 추가 |
| 색상 대비 | gray-400 (`#BDBDBD`) 텍스트가 white 배경에서 WCAG AA 미달 (대비 1.75:1, 최소 4.5:1 필요) | tertiary 텍스트를 gray-500 (`#9E9E9E`) 이상으로 변경 |
| 포커스 표시기 없음 | 키보드 사용자가 현재 위치를 알 수 없음 | `:focus-visible` 스타일 추가 (accent 색상 outline) |

### 1.4 관리자 앱 UX 원칙 미적용

스펙(§12)에서 정의한 **"IT 초등학생 수준 대응"** 원칙 중 일부 미반영:

| 원칙 | 현재 상태 | 개선 |
|------|----------|------|
| 자동화 극대화 | "업데이트 하기" 버튼을 수동으로 눌러야 카메라 목록 갱신 | 자동 갱신(10~30초 간격) + Pull-to-refresh |
| 오류 방지 | 녹화 중지 시 확인 없이 바로 진행 | "녹화를 중지할까요?" 확인 다이얼로그 |
| 온보딩 가이드 | 최초 실행 시 안내 없음 | 스펙 §12.4 온보딩 가이드 4단계 구현 필요 |

## 3. 디자인 시스템 & 비주얼

### 3.1 디자인 토큰 정합성

`styles.css`와 `DEV_SPEC.md`에 잘 정의된 디자인 시스템이 있으나, **실제 HTML에서 일관되게 사용되지 않는다.**

| 토큰 | 정의 | 실제 사용 | 문제 |
|------|------|----------|------|
| `--color-background` | `#F7F7F8` | 일부에서 `#FFFFFF` 직접 사용 | 불일치 |
| `--color-text-secondary` | `#757575` | `color: #757575` 직접 입력 | CSS 변수 미사용 |
| 간격 | `--space-4: 16px` 등 | `padding: 12px 16px` 직접 입력 | CSS 변수 미사용 |
| 보더 반지름 | `--radius-md: 12px` | `border-radius: 12px` 직접 입력 | CSS 변수 미사용 |

> **참고**: 이 프로토타입은 개발자 핸드오프용 단일 HTML 파일로, 인라인 스타일 사용은 프로토타입 특성상 허용된다. 다만 프로덕션 전환 시 디자인 토큰 기반 컴포넌트로 재구성해야 한다.

### 3.2 비주얼 일관성

| 영역 | 문제 | 개선 |
|------|------|------|
| **카드 그림자** | 토큰(`--shadow-md`)과 실제 사용 값 불일치 | 프로덕션에서 토큰 통일 |
| **아이콘 크기** | SVG 아이콘 크기가 화면마다 다름 (14px ~ 24px) | 아이콘 크기 토큰 정의 |
| **관리자 vs 소비자 구분** | 두 앱의 비주얼 차이가 불명확 | 관리자: 절제된 정보 중심, 소비자: 감성적 미디어 중심 차별화 |

### 3.3 모션 & 인터랙션

| 항목 | 현재 | 개선 |
|------|------|------|
| **페이지 전환** | 즉시 show/hide | 페이드 또는 슬라이드 전환 애니메이션 |
| **프레스 피드백** | `.card:active { transform: scale(0.98) }` → 잘 적용됨 | OK, 유지 |
| **stagger 애니메이션** | `fadeInUp` 정의됨, 일부 적용 | **모든 리스트 화면**에 일관 적용 |
| **Press & Hold** | 스펙 §4.2.3 "프레스앤홀드 → 5초 미리보기" | ✅ 구현 완료 (mouse + touch) |

### 3.4 콘텐츠 밀도

| 화면 | 현재 | 참고 |
|------|------|------|
| **ADM-01 (홈)** | 카메라 4대로 화면이 꽉 참 | 카메라가 10대 이상이면 구장별 그룹핑 필요 |
| **CON-07 (영상 상세)** | 영상 3종 + 선수 그리드가 세로로 나열 | 선수 그리드는 **인스타그램 피드 스타일 의도** — 세로 나열 유지 |
| **ADM-15 (일정)** | 캘린더 + 스케줄 목록이 한 화면에 | 캘린더 컴팩트 모드 (주간 뷰 기본) + 날짜 탭 시 하단 확장 |
