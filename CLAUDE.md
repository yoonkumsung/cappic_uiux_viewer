# CAPP!C 프로젝트 규칙

## 용어 규칙
- **전문 용어를 절대 사용하지 않는다.** IT 비전문가(축구교실 코치, 풋살장 운영자)가 바로 이해할 수 있는 쉬운 말로 써야 한다.
- 아래 매핑을 반드시 따른다:

| 금지 용어 | 사용할 용어 |
|----------|-----------|
| WebRTC | 실시간 화면 |
| metrics | 카메라 상태 |
| ReID | 선수 하이라이트 |
| discovery | 카메라 찾기 |
| firmware | 소프트웨어 |
| OTA update | 업데이트 |
| subnet scan | 자동 검색 |
| batch recording | 한꺼번에 녹화 |
| render | 영상 만들기 |
| threshold | 정확도 |
| concat | 영상 합치기 |
| upload / download | 올리기 / 내려받기 |
| stitch / stitching | 영상 합치기 / 영상 화질 |
| 사용자 (관리자앱) | 운영자 |
| 스티칭 출력 | 영상 화질 |
| 렌즈 촬영 | 촬영 화질 |
| score threshold | 점수 정확도 |
| match threshold | 매칭 정확도 |
| processing | 처리 중 |
| 중첩 비율 | 표시하지 않음 (불필요) |

- UI에 표시되는 모든 텍스트는 초등학생도 이해할 수 있는 수준으로 작성한다.
- 기술적 수치(중첩 퍼센트 등)는 사용자에게 노출하지 않는다. 상태만 보여준다 (정상/오류).

## 작업 규칙
- **지시한 사항은 하나도 빠짐없이 반영한다.** 여러 항목을 지시받으면 하나하나 차근차근 전부 처리한다.
- 빠뜨린 항목이 없는지 완료 후 반드시 체크한다.
- HTML 수정 시 div 태그 균형(open/close 개수)을 반드시 확인한다.
- 수정 후 기존 화면이 깨지지 않았는지 검증한다.
